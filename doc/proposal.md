# E-Commerce SmartOps Agent — 需求文档

## 1. 项目背景

面向跨境电商卖家（Shopify/Amazon 独立站）的 AI 智能运营助手。卖家日常需要分析
转化率、监控竞品、生成营销素材、管理知识库——当前依赖人工 + 多工具切换，效率低。

## 2. 目标用户

- **直接用户**：跨境电商运营人员（1-50 人团队）
- **使用方式**：Web SaaS，注册 → 获取 API Key → 使用
- **月活预期**：100-500 用户

## 3. 功能需求

### 3.1 运营仪表盘

| ID | 需求 | 优先级 |
|----|------|:---:|
| F1.1 | 日期联动的核心指标卡片（转化率、订单量、客单价、访客数） | P0 |
| F1.2 | 7 张交互式图表（趋势线、漏斗、饼图、柱状图、仪表盘、评分分布、订单柱图） | P0 |
| F1.3 | 支持用户自定义数据输入（覆盖 mock 数据） | P1 |

### 3.2 Agent 对话

| ID | 需求 | 优先级 |
|----|------|:---:|

## 4. 非功能需求

| ID | 需求 | 指标 |
|----|------|------|
| N1 | 首次对话响应 | < 10s（含 LLM 调用） |
| N2 | 缓存命中响应 | < 100ms |
| N3 | 并发用户 | 支持 50 并发请求 |
| N4 | 数据持久化 | SQLite + ChromaDB 本地文件 |
| N5 | 前端首屏加载 | < 3s（Vite + CDN） |
| N6 | 单元测试覆盖率 | ≥ 80% |
| N7 | 类型检查 | mypy strict + ruff lint |

## 5. 技术约束

- LLM: DeepSeek / OpenAI / Anthropic 三家统一接口
- Agent 框架: LangGraph（不变）
- 向量库: ChromaDB Embedded（不变）
- 数据库: SQLite（零运维，单文件）
- 后端: FastAPI + Uvicorn（已有基础）
- 前端: React + Vite + Ant Design（替换 Streamlit）
- 部署: Docker Compose（nginx + uvicorn + 静态文件）
- 测试: pytest + pytest-asyncio + pytest-cov
- 代码质量: mypy + ruff
